Guide d'Utilisation de la ligne de commande
Le Guide d'Utilisation de la version standard se trouve ici

Introduction
La version Ligne de commande de DeepSkyStacker est conçue pour simplifier les traitements batchs.
La version Ligne de commande de DeepSkyStacker utilise le même moteur d'enregistrement et d'empilement que la version standard de DeepSkyStacker. En d'autres termes vous aurez exactement les mêmes résultats en utilisant la version standard ou la version ligne de commande.

Pour le moment tous les paramètres sont partagés entre les versions standard et ligne de commande de DeepSkyStacker en particulier les paramètres RAW, les paramètres d'enregistrement et d'empilement, le répertoire des fichiers temporaires...

Ceci signifie en particulier qu'avant d'utiliser la version ligne de commande vous devez utiliser la version standard pour fixer tous les paramètres en fonction de vos souhaits.
Dans de futures versions il sera possible de fixer certains paramètres directement sur la ligne de commande.

Paramètres de la ligne de commande
La version ligne de commande de DeepSkyStacker est appelée en utilisant DeepSkyStackerCL.exe situé dans le répertoire DeepSkyStacker défini au moment de l'installation.

Une ligne de commande doit ressembler à :

DeepSkyStackerCL.exe [/r|R] [/S] [/SC] [/SR] [/O:<OutputFileName>] [/OFxxx] [/OCx] <ListFileName>

Où les paramètres sont :

/r
 		 Enregistre les images (uniquement si elles ne sont pas déjà enregistrées)
 
/R
 		 Enregistre les images (même si elles sont déjà enregistrées - équivalent de l'option forcer l'enregistrement)
 
/S
 		 Empile les images
 
/SC
 		 Empile les images et pour chacune d'elle crée un fichier contenant l'image calibrée.
 
/SR Empile les images et pour chacune d'elle crée un fichier contenant l'image enregistrée et calibrée.
 
/O:<OutputFileName>



 

 Nom optionnel du fichier de sortie (implique /S or /SC)
Par défaut le nom du fichier de sortie est Autosave.tif et il est situé dans le même répertoire que la première image.
Il faut noter que le fichier sera toujours un fichier TIFF même si l'extension est différente.
Le répertoire dans lequel le fichier de sortie sera situé doit exister.
 
/OFxxx



 		 Format du fichier de sortie.
Les valeurs possibles sont :
/OF16 or /OF16i : précision 16 bit entier
/OF32 or /OF32i : précision 32 bit entier
/OF32r : précision 32 bit réel (c'est la valeur par défaut)
 
/OCx



 		 Compression du fichier de sortie.
Les valeurs possibles sont :
/OC0 : aucune compression (c'est la valeur par défaut)
/OC1 : compression LZW
/OC2 : compression ZIP (deflate)
 
/FITS Enregistre le fichier de sortie au format FITS (le défaut est le format TIFF)
 
<ListFileName>

 		 Chemin complet vers un fichier contenant la liste de fichiers tel que créé par la version standard de DeepSkyStacker.
(voir les détails du format ci-dessous).

Exemples
DeepSkyStackerCL.exe /R c:\MyDSSLists\MyList.txt
Va enregistrer (y compris celles déjà enregistrées) les images de la liste c:\MyDSSLists\MyList.txt..
Si la liste contient des fichiers offsets/bias, dark et flat ils seront utilisés (et les masters seront créés) par le processus d'enregistrement.

DeepSkyStackerCL.exe /R /S c:\MyDSSLists\MyList.txt
Va enregistrer (y compris celles déjà enregistrées) puis empiler les images de la liste c:\MyDSSLists\MyList.txt.
Si la liste contient des fichiers offsets/bias, dark et flat ils seront utilisés (et les masters seront créés) par les processus d'enregistrement et d'empilement.
Le fichier de sortie sera un fichier TIFF 32 bits réels appelé Autosave.tif situé dans le même répertoire que la première image de la liste.

DeepSkyStackerCL.exe /r /SR c:\MyDSSLists\MyList.txt
Va enregistrer (uniquement celles non enregistrées) puis empiler les images de la liste c:\MyDSSLists\MyList.txt.
Si la liste contient des fichiers offsets/bias, dark et flat ils seront utilisés (et les masters seront créés) par les processus d'enregistrement et d'empilement.
Le fichier de sortie sera un fichier TIFF 32 bits réels appelé Autosave.tif situé dans le même répertoire que la première image de la liste.
De plus pour chaque image empilable un fichier TIFF ou FITS nommé d'après le nom de l'image avec une extension .reg.tif ou reg.fts sera créé. Ce fichier contient l'image enregistrée et calibrée.

DeepSkyStackerCL.exe /S /OF16i /OC2 /O:c:\MyDSSResults\Result.tif c:\MyDSSLists\MyList.txt
Va empiler toutes les images de la liste c:\MyDSSLists\MyList.txt..
Si la liste contient des fichiers offsets/bias, dark et flat ils seront utilisés (et les masters seront créés) par le processus d'empilement.
Le fichier de sortie sera un fichier TIFF 16 bits entier avec compression ZIP(DEFLATE) appelé c:\MyDSSResults\Result.tif.

Format d'une liste de fichiers
Une liste de fichiers est  fichier texte dont la structure est la suivante :

La première ligne doit contenir DSS file list
La deuxième ligne doit contenir CHECKED<TAB>TYPE<TAB>FILE
Où <TAB> est la caractère de tabulation (ASCII 09)

Les autres lignes doivent avoir la forme suivante si elles correspondent à un fichier :
[checked]<TAB>[type]<TAB>[full path name]

Où [checked] vaut 1 si l'image est cochée et 0 si elle n'est pas cochée.
(les images non cochées sont ignorées par DeepSkyStackerCL)
Où [type] vaut light pour une image brute (ou reflight pour l'image brute de référence), dark ou darkflat pour une image dark, flat pour une image flat et offset pour une image offset/bias.
Où [full path name] est le chemin complet vers le fichier.

Si une ligne correspond à un changement de groupe elle doit avoir la forme suivante :
#GROUPID#<Numéro du groupe>
Tous les fichiers situés après la ligne de changement de groupe et jusqu'à la prochaine ligne de changement de groupe seront affectés au groupe <Numéro du groupe>. <Numéro du groupe> doit être une valeur entière positive.
Lorsque aucun  groupe n'est précisé les fichiers sont affectés au groupe principal (identifiant 0).

Vous pouvez aisément créer des listes de fichier en utilisant la version standard de DeepSkyStacker (commande Enregistrer la liste de fichier ...).
