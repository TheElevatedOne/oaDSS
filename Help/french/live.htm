Introduction

DeepSkyStacker Live : c'est quoi ?
DeepSkyStacker Live est conçu pour être utilisé sur le terrain pendant une session pour les tâches suivantes :

  • Aligner et empiler les images au fur et à mesure qu'elles sont capturées
  • Calculer et montrer des statistiques sur les images (FWHM, nombre d'étoiles détectées, décalage...)
  • Montrer l'image résultant de l'empilement
  • Prévenir lorsque certaines conditions sont détectées, par exemple si le décalage est supérieur à une valeur choisie par l'utilisateur.

DeepSkyStacker Live est compatible avec toutes les caméras (APN, CCD) et avec tous les logiciels de captures à partir du moment où les images capturées sont enregistrées dans un répertoire et sont d'un format reconnue (FITS, TIFF, JPEG et images RAW de presque tous les APN).

Note : DeepSkyStacker Live ne remplace pas DeepSkyStacker.
Ils utilisent tous les deux le même moteur mais DeepSkyStacker Live ne possède pas les fonctionnalités permettant de créer des images pouvant être post traitées avec précision.
Par exemple DeepSkyStacker Live ne permet pas de calibrer les images (soustraction des darks et offset/bias, division par un flat) ou d'utiliser les méthodes d'empilement les plus avancées (kappa sigma ou moyenne pondérée adaptative).


Interface Utilisateur de DeepSkyStacker Live

Comme DeepSkyStacker l'interface utilisateur de DeepSkyStacker est simple et intuitive et toutes les fonctionnalités et commandes sont disponibles directement.


En haut de l'écran trois boutons sont utilisés pour démarrer la surveillance, démarrer l'empilement et arrêter..

La partie basse est utilisée pour surveiller l'avancement pendant la session de différentes manières :

  • L'image empilée

  • La dernière image

  • Des graphiques montrant diverses statistiques

  • La liste des images

  • Le journal

Le dernier onglet est utilisé pour fixer les paramètres permettant de contrôler le fonctionnement de DeepSkyStacker Live.

 

 

Démarrage Rapide
 Votre portable est prêt pour la session, le logiciel de capture CCD ou APN est lancé.

Démarrez DeepSkyStacker Live, cliquez sur le bouton Empiler et sélectionnez le répertoire à surveiller. C'est tout !

A partir de ce moment toutes les images créées dans le répertoire surveillé vont être enregistrées, alignées et empilées sauf si elles correspondent à une des conditions que vous avez fixées dans l'onglet Paramètres.

Bien entendu vous pouvez changer les paramètres pendant que DeepSkyStacker Live surveille l'arrivée de nouvelles images.

Paramètres
 Tout d'abord DeepSkyStacker Live partage tous les paramètres concernant le décodage des images, le seuil de détection des étoiles, les méthodes d'alignement et les options de calibration du fond avec DeepSkyStacker. Pour changer ces valeurs vous devez utiliser DeepSkyStacker.
La seule différence concerne les décodages AHD et Bayer Drizzle qui sont automatiquement transformés en Interpolation Bilinéaire.

L'onglet Paramètres est utilisé pour définir quand les nouvelles images sont empilées, quand un avertissement doit être envoyé et sous quelle forme.

Options d'empilement
Ne pas commencer l'empilement tant que xxx images ne sont pas disponibles
Cette option peut être utilisée pour attendre qu'un nombre minimum d'images soit disponible avant de décider quelle image sera utilisée comme image de référence.
Quand le nombre minimum d'image est atteint DeepSkyStacker Live utilise celle qui a le score le plus élevé comme image de référence.
Cette option est particulièrement utile si vous prenez des poses courtes et si vous n'êtes pas sur que toutes vos images sont assez bonnes.

Options d'Avertissement/Non empilement
Ces options s'expliquent d'elles mêmes et peuvent être utilisées pour définir diverses conditions pour lesquelles une image ne doit pas être empilée et/ou pour laquelle un avertissement doit être envoyé.
Il est possible de définir des conditions sur :

  • Le Score

  • Le Nombre d'étoiles détectées

  • La FWHM (Full Width Half Maximum)

  • Le décalage (dX ou dY)

  • L'angle de rotation

  • Le fond du ciel

Avertissements
Quand une condition d'avertissement est détectée il est possible de l'envoyer en utilisant 4 méthodes :

  • Son : le bip classique de l'ordinateur

  • Clignotement : DeepSkyStacker Live clignote 4 fois

  • Créer un fichier d'avertissement : un fichier nommé DDSLiveWarning.txt contenant le message d'avertissement est créé dans le répertoire de votre choix. Cela peut être un répertoire local ou sur le réseau. Si le fichier existe déjà le nouveau message d'avertissement est ajouté à la fin.

  • Envoyer un email : un email contenant le message d'avertissement est envoyée à l'adresse de votre choix.
    Dans la dialogue de configuration de l'email vous pouvez choisir de n'envoyer un email que pour le premier avertissement. Dans ce cas dès qu'un email a été envoyé un bouton apparaîtra vous permettant de remettre à zéro le compteur interne des emails envoyés.

Création de l'image empilée
L'image empilée est créée uniquement en mémoire et n'est pas enregistrée par défaut.
Cette option peut être utilisée pour créer un fichier contenant l'image empilée à chaque fois que xxx images sont ajoutées à la pile.

L'image empilée est enregistrée dans un fichier appelé Autostack.tif dans le répertoire de votre choix. Si le fichier existe déjà il est écrasé.
Il s'agit toujours d'une image TIFF 16 bits et en fonction de la nature des images capturées elle peut être en couleur ou niveau de gris.
 

Graphiques
 Pour chaque image DeepSkyStacker Live utilise les symboles suivants :

  • Un cercle vert pour chaque image

  • Un cercle orange derrière le cercle vert : une condition d'avertissement est rencontrée

  • Un triangle rouge par dessus le cercle vert : une condition de non empilement est rencontrée

  • Un carré vert est utilisé sur les graphiques dX/dY et Angles pour représenter l'image de référence

A propos du graphique dX/dY la ligne rouge représente dX et la ligne bleue représente dY.

Quand une image n'est pas empilable elle n'apparaît pas dans les graphiques dX/dY et Angle.

Informations Techniques
 DeepSkyStacker Live utilise l'empilement par la Moyenne (c'est la seule méthode possible lorsque le nombre d'images n'est pas connu à l'avance).
Comme aucune calibration n'est possible une simple détection et suppression des pixels chauds est utilisée sur les images entrantes avant de les empiler.
Vous pouvez voir les pixels chauds dans l'onglet Dernière Image mais il devraient être supprimés dans l'onglet Image Empilée.

Au contraire de DeepSkyStacker, DeepSkyStacker Live n'utilise qu'un processeur pour toutes les tâches de fond (chargement, enregistrement, empilement) même si plusieurs processeurs sont disponibles.
